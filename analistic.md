# Анализ ошибок при добавлении empty-line в главу 66

## Описание ошибки

При добавлении пустых строк (`<empty-line/>`) в главу 66 я допустил систематическую ошибку: добавлял `empty-line` между **всеми** последовательными блоками текста, независимо от их типа.

## Правило из template.md (раздел 2.2)

Согласно разделу 2.2 шаблона, пустые строки добавляются **ТОЛЬКО** при переходе между **разными** типами блоков:

1. **Диалог после прозы** — добавить `empty-line` **ПЕРЕД** диалогом
2. **Проза после диалога** — добавить `empty-line` **ПОСЛЕ** диалога
3. **Подзаголовок после прозы** — добавить `empty-line` **ПЕРЕД** подзаголовком
4. **Проза после подзаголовка** — добавить `empty-line` **ПОСЛЕ** подзаголовка
5. **Диалог после подзаголовка** — добавить `empty-line` **ПОСЛЕ** подзаголовка
6. **Подзаголовок после диалога** — добавить `empty-line` **ПЕРЕД** подзаголовком

## Моя ошибка

Я добавлял `empty-line` также при переходах:
- Диалог → Диалог (НЕ НУЖНО)
- Проза → Проза (НЕ НУЖНО)

## Примеры ошибочных добавлений в главе 66

### Ошибка 1: Диалог после диалога с empty-line

В главе 66 встречаются последовательные диалоги:
```
<p>— Не верю!</p>
<empty-line/>  ← ОШИБКА: не нужен, это диалог-to-диалог
<p>— Умоляю тебя, она совсем старенькая...</p>
```

Согласно правилам, между двумя диалоговыми блоками НЕ нужен разделитель.

### Ошибка 2: Лишние empty-line в конце диалоговых блоков

```
<p>— Пусть выдаст преступника...</p>
<empty-line/>  ← ПРАВИЛЬНО (диалог → проза)
<p>Су Су понимала: хищник...</p>
```

Это правильно, но далее:
```
<p>— Позволь мне с ней поговорить...</p>
<empty-line/>  ← ПРАВИЛЬНО (диалог → проза)
<p>Тан Тай Цзинь в ответ презрительно фыркнул:</p>
```

Это тоже правильно, но затем:
```
<p>— Мы не доверяем тебе.</p>
<empty-line/>  ← ПРАВИЛЬНО (диалог → проза)
<p>Более того, он был уверен...</p>
```

И так далее. Проблема в том, что я добавлял `empty-line` даже там, где диалог immediately переходит в описание (слова автора), которое является частью того же повествовательного потока.

## Проверка по чек-листу из template.md

Согласно чек-листу (раздел 2.4):
- [ ] Каждый `<subtitle>* * *</subtitle>` имеет `<empty-line/>` ДО и ПОСЛЕ — **ВЕРНО**
- [ ] Две пустые строки (`<empty-line/><empty-line/>`) могут быть ТОЛЬКО под названием главы — **ВЕРНО**
- [ ] Между разными блоками текста: блок прозы, диалоговый блок, подзаголовок - есть ровно одна `<empty-line/>` — **ЧАСТИЧНО ОШИБОЧНО**: добавлял лишние empty-line при диалог-to-диалог переходах
- [ ] Количество уникальных noteId... — **ВЕРНО**

## Рекомендация по исправлению

Необходимо:
1. Удалить все `empty-line`, которые находятся между двумя диалоговыми блоками (начинающимися с "— ")
2. Сохранить `empty-line` только при переходах диалог → проза и проза → диалог
3. Проверить все места, где диалог продолжается словами автора в том же абзаце — там не нужен разделитель

## Заключение

Моя ошибка заключалась в неправильной интерпретации правила 2.2. Я решил, что "между разными блоками" означает "между любыми последовательными блоками", но правило применяется только при смене типа блока (проз ↔ диалог ↔ подзаголовок), а не при продолжении того же типа.

# Анализ ошибок при преобразовании мыслей героев

## Правила из template.md (раздел 2.1.1.2)

Согласно разделу 2.1.1.2 шаблона:

1. **Мысли героя** — это текст, который начинается с символа "«" и заканчивается символом "»"
2. При преобразовании мыслей героя внутри "«" и "»" должен быть тег наклона `<emphasis>`
3. **Самы ковычки при этом переносяться в xml без изменений** (линия 115)

Пример правильного преобразования:
```
«Я ни за что не сдамся{знак_текста_мыслей}»
```
Должно стать:
```xml
<p>«<emphasis>Я ни за что не сдамся{знак_текста_мыслей}</emphasis>»</p>
```

Важно: открывающая кавычка "«" должна быть **ВНЕ** тега `<emphasis>`, а закрывающая "»" также **ВНЕ** тега `<emphasis>`.

## Обнаруженные ошибки в главе 66

### Ошибка 1: Мысли с внутренними словами автора (линия 12104)

**Исходный текст:**
```xml
<p><emphasis>Я всегда была хорошей и послушной</emphasis>, — думала Бин Чан, глядя в испуганные глаза служанки.</p>
```

**Проблема:** Тег `<emphasis>` охватывает "Я всегда была хорошей и послушной" и запятую, но НЕ включает открывающую кавычку "«" и закрывающую "»" — они отсутствуют в тексте.

Согласно шаблону, правильный формат:
```xml
<p>«<emphasis>Я всегда была хорошей и послушной</emphasis>», — думала Бин Чан...</p>
```

Открывающая кавычка должна быть **до** `<emphasis>`, закрывающая — **после** `</emphasis>`.

### Ошибка 2: Мысли с внутренними словами автора (линия 12105)

**Исходный текст:**
```xml
<p><emphasis>Интересно, он уже решил, как накажет третью сестру? Эта мерзавка не только пыталась атаковать его величество, но ещё и ранила императорскую супругу. Он точно не пощадит её на этот раз</emphasis>, — Бин Чан прикрыла рот ладонью...</p>
```

**Проблема:** Та же — кавычки отсутствуют, `<emphasis>` охватывает неправильную часть.

Согласно шаблону, мысль должна быть:
```
«Интересно, он уже решил, как накажет третью сестру? Эта мерзавка не только пыталась атаковать его величество, но ещё и ранила императорскую супругу. Он точно не пощадит её на этот раз»
```

И преобразование:
```xml
<p>«<emphasis>Интересно, он уже решил, как накажет третью сестру? Эта мерзавка не только пыталась атаковать его величество, но ещё и ранила императорскую супругу. Он точно не пощадит её на этот раз</emphasis>» — Бин Чан прикрыла рот ладонью...</p>
```

## Правило для мыслей с внутренними словами автора

Согласно примеру из шаблона (линия 125-128):
```
«Я ни за что не сдамся{знак_текста_мыслей} — подумала она{знак_слов_автора} — Где бы ты ни был - я тебя найду{знак_текста_мыслей}»
```
Преобразуется в:
```xml
<p>«<emphasis>Я ни за что не сдамся{знак_текста_мыслей}</emphasis> — подумала она{знак_слов_автора} — <emphasis>Где бы ты ни был - я тебя найду{знак_текста_мыслей}</emphasis>»</p>
```

То есть:
- Кавычки "«" и "»" остаются **за пределами** `<emphasis>`
- Слова автора ("— подумала она") находятся **между** `<emphasis>` тегами, без кавычек
- Мысли героя оборачиваются в `<emphasis>`, но кавычки к ним не относятся

## Заключение по мыслям

В главе 66 при преобразовании мыслей героев были допущены следующие ошибки:
1. Отсутствуют кавычки "«" и "»" в XML — они должны быть добавлены согласно шаблону
2. Тег `<emphasis>` неправильно охватывает мысли — должен быть только вокруг текста мысли, без слов автора внутри

# Дополнительные проблемы, выявленные при детальной проверке главы 66

## Проблема 1: Непоследовательные empty-line после диалога с "—"

В главе 66 присутствуют случаи, когда после диалога (строки, начинающиеся с "— ") ставится `<empty-line/>`, даже если далее следует описание действий персонажа (слова автора), логически связанное с предыдущим повествованием.

Пример:
```
<p>— Если ты, как и старшая внучка, влюбилась в этого негодяя, не смей называй меня своей бабушкой!</p>
<empty-line/>
<p>— Глаза Су Су наполнились слезами...
```

Правило 2.2 требует `<empty-line/>` только при смене типа блока. Когда после диалога идёт описание реакции персонажа (слова автора), логически связанное с диалогом — это не смена типа блока, а продолжение повествовательного потока.

## Проблема 2: Возможные лишние empty-line в последовательности "диалог → empty-line → описание"

В ряде мест после диалога следует описание, которое логически является частью того же эпизода, но между ними стоит `<empty-line/>`. Хотя формально это не ошибка (прозу можно отделять от диалога), это может нарушать принцип "не разбивать текст на абзацы, если в оригинале он был единым блоком" ( правило 2.0).

## Рекомендация

При финальной проверке главы 66 необходимо:
1. Проверить каждый случай использования `<empty-line/>` после диалога
2. Определить, является ли следующий за диалогом текст логическим продолжением (слова автора в том же эпизоде) или новым блоком
3. Удалить `<empty-line/>` там, где диалог плавно переходит в описание реакции персонажа без смены темы/эпизода
