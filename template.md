# 1. Шаблон книги
```
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE FictionBook>
<FictionBook xmlns="http://www.gribuser.ru/xml/fictionbook/2.0" xmlns:l="http://www.w3.org/1999/xlink">
    <description>
        <title-info>
        </title-info>
        <document-info>
         </document-info>
        <publish-info>
        </publish-info>
    </description>
<!-- Parts -->
     <body>
    <!-- Book {BookNum}. {Название книги} -->
         <!-- Part {PaptNum} «{Название тома книги}» -->
            <section>
                <title><p><strong>Часть {PaptNum}. «{Название тома книги}»</strong></p></title>
                <empty-line/>
                <image l:href="#part_{PaptNum}.jpg" />
                <empty-line/>
                <epigraph>
                    <p>«{Хорошая цитата из этой части}»</p>
                    <empty-line/>
                    <text-author>{Автор этой цитаты}</text-author>
                    </epigraph>
            <!-- Chapter {ChapterNum} -->
                <section>
                    <title><p><strong>Глава {ChapterNum}. «{Название главы}»</strong></p></title>
                    <empty-line/>
                    <empty-line/>
                    <p>{Слова автора}<p>
                    <empty-line/>
                    <p>— {Диалог между словами автора}<p>
                    <empty-line/>
                    <p>{Слова автора}<p>
                    <empty-line/>
                    <p>— {Диалог из нескольких реплик}<p>
                    <p>— {Диалог из нескольких реплик}<p>
                    <empty-line/>
                    <p>{Слова автора}<p>
                    <empty-line/>
                    <image l:href="#{ChapterNum}_{название_изображения}" />
                    <empty-line/>
                    <p>{Слова автора}<p>
                    <p>{слова автора с словом, для которого есть примечание}<a l:href="#note{ChapterNum}_{NoteNum}" type="note">[{NoteNum}]</a></p>
                    <empty-line/>
                    <subtitle>* * *</subtitle>
                    <empty-line/>
                    <p><emphasis>{Мысли героев должны быть в наклоне}</emphasis><p>
                </section>
            </section>
    </body>
<!-- Notes -->
    <body name="notes">
    <!-- Book {BookNum}. {Название книги} -->
        <!-- For Part {PaptNum} -->
            <!-- For Chapter {ChapterNum} -->
                <section id="note{ChapterNum}_{NoteNum}">
                    <p><strong>{Определяемое слово}</strong> – {описание}.</p>
                    <empty-line/>
                    <image l:href="#{ChapterNum}_{название_изображения}" />
                    <empty-line/>
                    </section>
    </body>
<!-- Images -->
     <!-- Book {BookNum}. {Название книги} -->
        <!-- For Part {PaptNum} -->
            <binary id="part_{PaptNum}.jpg" content-type="image/jpeg">
                {Base64 image}
                </binary>
            <!-- For Chapter {ChapterNum} -->
                <binary id="{ChapterNum}_{название изображения}" content-type="image/jpeg">
                    {Base64 image}
                    </binary>
                <!-- For Chapter {ChapterNum} - notes -->
                    <binary id="{ChapterNum}_{название изображения}" content-type="image/jpeg">
                        {Base64 image}
                        </binary>
</FictionBook>
```

# 2. Примечание к работе
- Оригинал работы находится в @`Когда луна догорит дотла.docx`
- Нумерация частей и глав - сквозная, то есть в если в первой книге 3 части и главы от 1-ой до 10-ой, то во второй книге с 4-ой по 6-ую части и главы с 11-ой по 20-ую (в качесвте примера)
- На данный момент книга преобразованна в xml до 80 главы, включительно. В @`Светлый пепел луны.xml` подготовлено место для преобразования глав вплоть до 89 главы, включительно, a в @`Когда луна догорит дотла.docx` уже есть текст до 88 главы, включительно.
- Не обращай внимание на файлы @notes.xml и @images.xml и не меняй их.
- Комментарии, например "<!-- Images -->", которые есть в шаблоне обзательны
- Главы, картинки и note добавляются в файл @`Светлый пепел луны.xml`

### 2.0.1 Сохранение границ абзацев и текста
- Оригинальный docx файл содержит текст с переносами строк, которые определяют границы абзацев
- При split по ' — ' или другим разделителям НИКОГДА не объединять соседние абзацы в один
- Если в оригинале после "— Диалог." был отдельный абзац "Слова автора", то после преобразования это должны быть ДВА отдельных <p> тега
- Каждый абзац в оригинале = отдельный <p> в XML (кроме случаев объединения слов автора с диалогом в одну строку по правилам диалогов)

### 2.0.2 ЗАПРЕТ НА ИЗМЕНЕНИЕ ТЕКСТА
- **НИКОГДА не изменять, не пересказывать и не придумывать текст** при преобразовании из DOCX в XML
- Текст должен быть скопирован дословно, включая:
  - Все слова и предложения
  - Пунктуацию (точки, запятые, кавычки и т.д.)
  - Регистр букв
  - Переносы строк в оригинале
- Единственное допустимое преобразование - добавление XML тегов (`<p>`, `<emphasis>`, `<a>` и т.д.) по правилам шаблона
- Если текст в оригинале был одним абзацем, он должен остаться одним абзацем в XML
- Если текст содержит примечание (начинается с `*`), ссылку на него (слово с `*`) или изображение (имя файла .jpg) - это сохраняется как есть

## 2.1 Основные блоки и их определения в тексте и преобразования в xml:

### 2.1.1 Блок прозы
- *Прозой* называются абзацы с текстом, не начинающиеся на `— `.
- *Проза* - это именно абзац, то есть текст, расположенный на одной строке. Любой текст на следующей строке после *прозы* - это уже отлельный текст.

#### 2.1.1.1 Слова автора
- *Слова автора* - это предложение в тексте.
- Знаком припинания в конце *слов автора* может быть: `.`, `!`, `?`, `...`, `:` (обозначим их "знак_слов_автора").
- Если знака припинания нет вовсе - это скорее всего Ошибка и там должен быть символ `.`.
Например: `Ли Су Су долго бежала по лесу{знак_слов_автора}`, где все это предложение - *слова автора*
Преобразование в xml: ```<p>Ли Су Су долго бежала по лесу{знак_слов_автора}</p>```

#### 2.1.1.2 Мысли героя
- *Мысли героя* - это текст, который начинается с символа `«`, а заканчивается символом `»`, включительно.
- *Текст мыслей* - это преложение внутри *мыслей героя*.
- Знаком припинания в конце *текста мыслей* может быть: `,` `.`, `!`, `?`, `...` или не иметь знака (``) (обозначим их "знак_текста_мыслей").
- При преобразовании *мыслей героя* внутри `«` и `»` должен быть тег наклона (```«<emphasis>{мысли героя}</emphasis>»```).
- Сами ковычки при этом переносяться в xml без изменений
Например: `«Я ни за что не сдамся{знак_текста_мыслей}»`, где `Я ни за что не сдамся{знак_текста_мыслей}` - это *текст мыслей*.
Преобразование в xml: ```<p>«<emphasis>Я ни за что не сдамся{знак_текста_мыслей}</emphasis>»</p>```

1) *Мысли героя* могут состоять из нескольких *текстов мыслей*.
Например: `«Я ни за что не сдамся{знак_текста_мыслей} Где бы ты ни был - я тебя найду{знак_текста_мыслей}»`, где `Я ни за что не сдамся{знак_текста_мыслей}` и `Где бы ты ни был - я тебя найду{знак_текста_мыслей}` - это два *текста мыслей*.
Преобразование в xml: ```<p>«<emphasis>Я ни за что не сдамся{знак_текста_мыслей} Где бы ты ни был - я тебя найду{знак_текста_мыслей}</emphasis>```

2) *Мысли героя* могут прерываться на *слова автора* при помощи знака ` — `;
Примечание: *слова автора* при этом не являются *мыслями героя* и не должны находиться внутри тегов наклона.
Например: `«Я ни за что не сдамся{знак_текста_мыслей} — подумала она{знак_слов_автора} — Где бы ты ни был - я тебя найду{знак_текста_мыслей}»`, где:
    `Я ни за что не сдамся{знак_текста_мыслей}` и `Где бы ты ни был - я тебя найду{знак_текста_мыслей}` - это два *текста мыслей*,
    а `подумала она{знак_слов_автора}` - это *слова автора*.
Преобразование в xml: ```<p>«<emphasis>Я ни за что не сдамся{знак_текста_мыслей}</emphasis> — подумала она{знак_слов_автора} — <emphasis>Где бы ты ни был - я тебя найду{знак_текста_мыслей}</emphasis>»</p>```

3) *Мысли героя* могут прерываться на *слова автора* при помощи:
    знака ` — ` для отделения *слова автора*, следующих за *текстом мыслей*,
    а затем знака `: ` для отделения *текста мыслей*, следующих за *словами автора* с пустым знаком на конце (``);
Например: `«Я не этого хотел{знак_текста_мыслей} — решил он и затем подумал: Я всё исправлю{знак_текста_мыслей}»`,
где:
    `Я не этого хотел{знак_текста_мыслей}` и `Я всё исправлю{знак_текста_мыслей}` - это два *текста мыслей*,
    а `решил он и затем подумал` - *это слова автора* без знака припинания на конце.
Преобразование в xml: ```<p>«<emphasis>Я не этого хотел{знак_текста_мыслей}</emphasis> — решил он и затем подумал: <emphasis>Я всё исправлю{знак_текста_мыслей}</emphasis>»</p>```

#### 2.1.1.3 Состав прозы
- *Проза* может состоять из:
1) одного или более *слов автора*.
Например: `Ли Су Су долго бежала по лесу{знак_слов_автора} Когда она нашла его, то крикнула{знак_слов_автора}`,
    это *проза*, состоящая из 2-х *слов автора*: `Ли Су Су долго бежала по лесу{знак_слов_автора}` и `Когда она нашла его, то крикнула{знак_слов_автора}`
Преобразование в xml: ```<p>Ли Су Су долго бежала по лесу{знак_слов_автора} Когда она нашла его, то крикнула{знак_слов_автора}</p>```

2) одной *мысли героя*.
Например: `«Я ни за что не сдамся{знак_текста_мыслей}»`,
где:
    «Я ни за что не сдамся{знак_текста_мыслей}» - это *мысли героя*,
    состоящие из одного *текста мыслей*: `Я ни за что не сдамся{знак_текста_мыслей}`.
Преобразование в xml: ```<p>«<emphasis>Я ни за что не сдамся{знак_текста_мыслей}</emphasis>»</p>```

3) *Мыслей героя*, следующих за *словами автора*.
Например: `Она бежала, не оборачиваясь{знак_слов_автора} «Я ни за что не сдамся{знак_текста_мыслей}»`,
где:
    `Она бежала, не оборачиваясь{знак_слов_автора}` - это слова автора,
    а `Я ни за что не сдамся{знак_текста_мыслей}` - это *мысли героя*,
    состоящие из одного *текста мыслей*: `Я ни за что не сдамся{знак_текста_мыслей}`.
Преобразование в xml: ```<p>Она бежала, не оборачиваясь{знак_слов_автора} «<emphasis>Я ни за что не сдамся{знак_текста_мыслей}</emphasis>»</p>```

4) *Мыслей героя*, следующих за *словами автора* без знаков припинания при помощи соединительного `: `.
Например: `Она не могла не признать самой себе: «Я боюсь{знак_текста_мыслей}»`,
где:
    `Она не могла не признать самой себе` - слова автора без знака припинания на конце,
    а `«Я боюсь{знак_текста_мыслей}»` - это *мысли героя*,
    состоящие из одного *текста мыслей*: `Я боюсь{знак_текста_мыслей}`.
Преобразование в xml: ```<p>Она не могла не признать самой себе: «<emphasis>Я боюсь{знак_текста_мыслей}</emphasis>»</p>```

5) *Слов автора*, следующих за *мыслями героя*.
Например: `«Зачем мне все это{знак_текста_мыслей} — подумал он{знак_слов_автора} — Уж лучше закончить всё здесь{знак_текста_мыслей}» Эти мысли не оставляли его в покое{знак_слов_автора}`,
где:
    `«Зачем мне все это{знак_текста_мыслей} — подумал он{знак_слов_автора} — Уж лучше закончить всё здесь{знак_текста_мыслей}»` - это *мысли героя*,
    состоящие из нескольких *текстов мыслей*: `Зачем мне все это{знак_текста_мыслей}` и `Уж лучше закончить всё здесь{знак_текста_мыслей}`,
    а так же *слов автора*, прерывающих *мысли героя*: `подумал он{знак_слов_автора}`,
    и еще одних *слов автора*: `Эти мысли не оставляли его в покое{знак_слов_автора}`
Преобразование в xml: ```<p>«<emphasis>Зачем мне все это{знак_текста_мыслей</emphasis> — подумал он{знак_слов_автора} — <emphasis>Уж лучше закончить всё здесь{знак_текста_мыслей}</emphasis>»</p>```

#### 2.1.1.4 Примечания
- *Примечания* - это некоторый текст, содержащий некое пояснение или определение какого-то слова или термина.
- *Примечания* помогают читателю лучше понять контекст.
- *Примечания* распологаются под каждой главой в тексте @`Когда луна догорит дотла.docx`, где такие *примечания* были нужны.
- Каждое *примечание* начинается с одного или более символов `*`.
- Если примечаний у главы несколько, то каждое следующее *примечание* начинается на то количество символов `*`, которым по счету примечанием оно является.
Пример 1: `*Консумация — это закрепление брака брачной ночью.`
где `Консумация — это закрепление брака брачной ночью` - это первое *примечание* главы.
Пример 2: 
```
*Консумация — это закрепление брака брачной ночью.
**Сюаньи — малая церемониальная одежда императора, черно-красного цвета.
```
где `Консумация — это закрепление брака брачной ночью` - это первое *примечание* главы, а `Сюаньи — малая церемониальная одежда императора, черно-красного цвета.` - это второе *примечание* главы.

- Каждое *примечание* к главе должно распологаться внизу файла  @`Светлый пепел луны.xml` в разделе <body name=`notes`> в блоке, относящемся к той главе, в которой это *примечание* распологается.
- На каждое *примечание* должна быть *ссылка на примечание* в тексте.
- *Ссылка на примечание* - это слово, за которым следует то количество симсолов `*`, которое равняется тому количеству `\*`, на которое оно ссылается.
- При этом звезды у *ссылок на примечание* наменяются тегом <a>, а количество звезд пишется в `[{количество_звезд}]` внутри этой ссылки
Пример 1: 
`*Консумация — это закрепление брака брачной ночью.` - если это примечание.
То ссылка в тексте будет, например: `консумация*`.
Преобразование ссылки на примечание в xml: ```консумация<a l:href="#note{номер_главы}_1" type="note">[1]</a>```

Пример 2: 
`**Сюаньи — малая церемониальная одежда императора, черно-красного цвета.` - если это примечание.
То ссылка в тексте будет, например: `в белой сюаньи**`.
Преобразование ссылки на примечание в xml: ```в белой сюаньи<a l:href="#note{номер_главы}_2" type="note">[2]</a>```

##### Преобразование примечание в xml и ссылок на них:
Пример: 
После главы N располагается 2 *примечания*: `*Консумация — это закрепление брака брачной ночью.` и `**Сюаньи — малая церемониальная одежда императора, черно-красного цвета.`
В тексте главы N есть 2 *ссылки на примечания*: `консумация*` и `в белой сюаньи**`
1) Преобразование *примечаний* и его расположение:
```

                    </section>

                    <p>Сюаньи — малая церемониальная одежда императора, черно-красного цвета.</p>

```
2) Преобразование *ссылок на примечания* в xml:
```консумация<a l:href="#note{N}_1" type="note">[1]</a>``` и ```в белой сюаньи<a l:href="#note{N}_2" type="note">[2]</a>```

### 2.1.2 Диалоговый блок
- *Диалогом* называются абзацы с текстом, который всегда начинающейся с `— `.
- *Диалог* - это именно абзац, то есть текст, расположенный на одной строке. Любой текст на следующей строке после диалога - это уже отлельный текст.
- *Текст диалога* - это предложение, расположенное внутри *диалога*, может быть двух видов:
1) Следует за `— ` и заканчивается на: `,` `.`, `!`, `?`, `...` (обозначим их "знак_текста_диалога").
Например: `— Меня зовут Су Су{знак_текста_диалога}`, где `Меня зовут Су Су{знак_текста_диалога}` - это *текст диалога*.
2) Следует за `: ` и заканчивается на `{знак_текста_диалога}`.
Например: `: Я уже это знаю{знак_текста_диалога}`, где `Я уже это знаю.` - это *текст диалога*.
Преобразование в xml в этих двух случаях будут рассмотрены ниже в `Видах диалога`
- если знака припинания после *текста диалога* нет вовсе - это скорее всего Ошибка и там должен быть символ `.`;
- *Диалог* может состоять из одного или более *текстов диалога*.
Например: `— Меня зовут Су Су{знак_текста_диалога} Я пришла сюда, чтобы тебя спасти{знак_текста_диалога}`,
где: `Меня зовут Су Су{знак_текста_диалога}` и `Я пришла сюда, чтобы тебя спасти{знак_текста_диалога}` - это два *текста диалога*.
Преобразование в xml: ```<p>— Меня зовут Су Су{знак_текста_диалога} Я пришла сюда, чтобы тебя спасти{знак_текста_диалога}</p>```

- В *Диалогах* могут встречаться *слова автора* по определенным правилам.
Рассмотрим ниже все виды диалогов
#### Виды диалогов:
##### 2.1.2.1. Диалог из текста диалога без слов автора
Примечания:
- вначале такого *диалога* всегда: `— `;
Примеры:
1) *Диалог* из одного *текста диалога*
Пример в тексте: `— Я - твой император{знак_текста_диалога}`
Преобразование в xml: ```<p>— Я - твой император{знак_текста_диалога}</p>```

2) *Диалог* из нескольких *текстов диалога* (от 1-ого до N):
Пример в тексте: `— Я ничего такого не помню{знак_текста_диалога} И мне всё равно{знак_текста_диалога}`
Преобразование в xml: ```<p>— Я ничего такого не помню{знак_текста_диалога} И мне всё равно{знак_текста_диалога}</p>```

##### 2.1.2.2. Диалог со словами автора, следующимиза текстом диалога
Примечания:
- вначале такого *диалога* всегда: `— `;
- для перехода к *словам автора* в *диалоге* после `{знак_текста_диалога}` ставится `— `, а затем следуют *слова автора*;
- *слова автора* в *диалоге* могут начинаться как с заглавной, так и с прописной буквы;
Пример в тексте: `— Ничего подобного{знак_текста_диалога} — возмутился он{знак_слов_автора}`
Преобразование в xml: ```<p>— Ничего подобного{знак_текста_диалога} — возмутился он{знак_слов_автора}</p>```

##### 2.1.2.3. Составной диалог со словами автора, прерывающими текст диалогов
Примечания:
- вначале такого *диалога* всегда: `— `;
- для перехода к *словам автора*, следующим за *текстом диалога*, после `{знак_текста_диалога}` ставится `— `, а затем следуют *слова автора*;
- *слова автора* в таком *диалоге* могут начинаться как с заглавной, так и с прописной буквы;
- следующие *текста диалога* могут начинаться как с заглавной, так и с прописной буквы;
- а для перехода от *словам автора* к следующему *тексту диалога*, после `{знака_слов_автора}` может быть: `— ` или `: `.
1) Знак `— `.
Пример в тексте: `— Я не хочу{знак_текста_диалога} — она немного запнулась{знак_слов_автора} — Я не могу{знак_текста_диалога}`
Преобразование в xml: ```<p>— Я не хочу{знак_текста_диалога} — она немного запнулась{знак_слов_автора} — Я не могу{знак_текста_диалога}</p>```

2) Знак `: `, когда *слова автора* заканчиваются без знака преринания.
Пример в тексте: `— Это не{знак_текста_диалога} — она прервалась{знак_слов_автора} Но в следующий миг передумала: — Это совсем не важно{знак_текста_диалога}`
Преобразование в xml: ```<p>— Это не{знак_текста_диалога} — она прервалась{знак_слов_автора} Но в следующий миг передумала: — Это совсем не важно{знак_текста_диалога}</p>```

- такой диалог может также заканчиваться словами автора.
Пример в тексте: `— Меня не устаивает такой подход{знак_текста_диалога} — сказала она{знак_слов_автора} — Так что не говори мне об этом{знак_текста_диалога} — развернулась и ушла прочь{знак_слов_автора}`
Преобразование в xml: ```<p>— Меня не устаивает такой подход{знак_текста_диалога} — сказала она{знак_слов_автора} — Так что не говори мне об этом{знак_текста_диалога} — развернулась и ушла прочь{знак_слов_автора}```

### 2.1.3 Подзаголовок
- *Подзаголовок* - это встречающиеся в тексте абзац из трех звед - `***`
- Он нужен для разбиения главы на некоторые эпизоды.
Пример: `***`
Преобразование в xml: ```<subtitle>* * *</subtitle>```

### 2.1.4 Изображение
- Изображения встречаются в тексте как имя файла в формате: {номер главы}_{имя}.{jpg или jpeg}
- Эти изображения расположены в папке `resources/Part_{часть, в которой расположена глава {номер главы}}/`
- Для каждого изображения нужно:
1) Получить base64 самого изображения (обозначим как "BASE64")
Получение:
```
#bash
base64 -i resources/Part_{часть, в которой расположена глава {номер главы}}/{имя изображения}
```
2) Добавить это этот base64 в `<binary>` тэг в специальный раздел
```
<!-- Images -->
    <!-- Book {номер книги, в которой расположена глава {номер главы}} {название книги} -->
        <!-- For Part {часть, в которой расположена глава {номер главы}} -->
            <!-- For Chapter {номер главы} -->
                <binary id="{номер главы}_{имя}.{jpg или jpeg}" content-type="image/jpeg">
                    {BASE64}
                    </binary>
```
3) Преобразовать упоминание картинки в тексте главы, `{номер главы}_{имя}.{jpg или jpeg}` в xml: `<image l:href="#{номер главы}_{имя}.{jpg или jpeg}" />`

Пример:
В книге 1, части 4, главе 18 в тексте есть строка: `18_e_si_yu.jpg`
1) Полученный base64 для файла `resources/Part_4/18_e_si_yu.jpg` запизывается в:
```
<!-- Images -->
    <!-- Book 1 Прошлое повелителя демонов -->
        <!-- For Part 4 -->
            <!-- For Chapter 18 -->
                <binary id="18_e_si_yu.jpg" content-type="image/jpeg">
                    {BASE64}
                    </binary>
```
2) строка `18_e_si_yu.jpg` преобразуется в `<image l:href="#18_e_si_yu.jpg" />`

## 2.2 Разделение основных блоков текста: прозы, диалогов и подзаголовка, пустыми строками
- После разбиения всего текста на блоки нужно разделить их *пустой строкой*, то есть `<empty-line/>`, при переходе от одного типа блока к другому.
- Главное правило: две подряд идущие *пустые строки* в может быть ТОЛЬКО под названием главы
Пример:
```
Глава {Номер главы}. «{Название главы}»


```
Преобразовывается в:
```
<title><p><strong>Глава {Номер главы}. «{Название главы}»</strong></p></title>
<empty-line/>
<empty-line/>
```
- Во всех остальных случаях - две *пустые строки* друг за другом - это ошибка.
- *Пустые строки* должны быть добавлены между разными типами блоков в случаях, если:
1) Текущая строка это *Подзаголовок* - тогда ДО и ПОСЛЕ должна быть `<empty-line/>`, то есть:
```
<empty-line/>
<subtitle>* * *</subtitle>
<empty-line/>
```
2) Текущая строка - это *изображение* - тогда ДО и ПОСЛЕ должна быть `<empty-line/>`, то есть:
```
<empty-line/>
<image l:href="#18_e_si_yu.jpg" />
<empty-line/>
```
3) Текущая строка - это *диалоговый блок*, то есть начинается с `— `, а следующая строка - это *блок прозы* или *мысль героя*, то есть начинается НЕ с `— `, то есть:
```
<p>— Ничего подобного{знак_текста_диалога} — возмутился он{знак_слов_автора}</p>
<empty-line/>
<p>Он развернулся и ушел прочь{знак_слов_автора}</p>
```
4) Текущая строка - это *блок прозы* или *мысль героя*, то есть начинается НЕ с `— `, а следующая строка - это *диалоговый блок* (начинается с `— `), то есть:
```
<p>«<emphasis>Я ни за что не сдамся{знак_текста_мыслей}</emphasis>» — подумал он{знак_слов_автора}</p>
<empty-line/>
<p>— Остановись{знак_текста_диалога} — кричала ему в след Е Си У{знак_слов_автора}</p>
```

#### 2.4.1 Проверки для каждой строки:
